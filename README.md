# Warehouse Management System

## System Description
This project is a simple object-oriented design for a Warehouse Management System.

An organization may have multiple warehouses. Each warehouse has one storekeeper. The storekeeper is responsible for handling goods in that warehouse.

## Functional Requirements

### 1. Goods Receipt
The storekeeper can receive goods into the warehouse.
When receiving goods, a goods receipt slip is created and printed.
The receipt slip contains:
- received products
- quantities
- person who delivered the goods
- received date

### 2. Goods Issue
The storekeeper can issue goods from the warehouse.
When issuing goods, a goods issue slip is created and printed.
The issue slip contains:
- issued products
- quantities
- person who received the goods
- issued date

### 3. Inventory Report
The storekeeper can view inventory reports at any time.
The report is generated by selecting:
- start date
- end date

The report shows for each product:
- beginning balance
- total receipt
- total issue
- ending balance

The report can also be filtered by one or more selected products.

### 4. Stock Count
The storekeeper can perform stock count for one selected product.
The stock count records:
- count date
- actual quantity

If the actual quantity differs from the expected quantity:
- surplus is recorded if actual > expected
- shortage is recorded if actual < expected

After stock count, the inventory quantity is updated according to the count date.
